# v1.0.0 — First Production Release

_Draft · 2026-07-03_

The first production release of **Fundamentals** — an end-to-end equity-research platform that
ingests SEC EDGAR filings for ~3,000 US companies, derives comparable financial metrics and
intrinsic-value estimates, backtests investment archetypes, and serves it all through a public
dashboard and an authenticated web application.

This release marks the point where every layer — the Databricks pipeline, the shared analytical
library, the public Streamlit dashboard, and the Django web app + REST API — is feature-complete,
tested, documented, and ready to deploy.

---

## Highlights

- **Filing-to-figure pipeline.** SEC EDGAR XBRL (10-K/10-Q) ingested and joined to market prices,
  normalized into comparable metrics, valued four ways, and backtested — no hand-curation.
- **Intrinsic value, four methods × three scenarios.** Graham, DCF, Owner Earnings, and EPS-CAGR,
  each under bull / mid / bear assumptions, with a margin of safety versus the market.
- **A single source of truth.** All financial logic and the data contract live in one installable
  Python package (`fundamentals_pipeline`), imported identically by the pipeline, the tests, the
  Streamlit app, and the web app — never reimplemented.
- **Two front doors.** A public Streamlit dashboard, and a richer Django web app with accounts,
  watchlists, and a versioned REST API.
- **Production-ready.** Hardened production settings, a Fly.io deployment, health probes,
  structured logging + error tracking, and a green CI gate (tests + types + lint + coverage floor).

---

## The analytical pipeline

- **SEC ingestion** — parallel, rate-limited fetch of each company's own XBRL from EDGAR, joined
  to Yahoo Finance prices.
- **Normalization** — annual merge with balance-sheet dedup, and quarterly derivation where
  `Q4 = FY − YTD_Q3` (capturing year-end audit adjustments), so a fiscal-year total always ties
  back to what the company filed.
- **Period-end-aligned pricing** — each fiscal year's market cap is priced at the company's real
  fiscal close, so non-December filers (e.g. Apple/Sep, Microsoft/Jun) aren't mispriced against a
  calendar year-end (`market_cap_asof`).
- **Intrinsic value** — Graham (with per-ticker trailing EPS-CAGR growth), DCF, Owner Earnings, and
  EPS-CAGR, each in bull / mid / bear, with margin of safety.
- **Backtester** — investment-archetype screens replayed across history strictly as-of each date
  (no look-ahead), reporting CAGR, drawdown, volatility, and Sharpe.
- **Data-quality work** — cash-flow sign/statement fixes, split-adjusted cross-year metrics,
  short-term-debt aggregation, goodwill/tangible ratios, and concept-mapping coverage improvements.

## The shared library & data contract

- `fundamentals_pipeline` is an **installable package** (`pip install -e .`) — no `sys.path` hacks.
  Its pure-Python modules (`schemas`, `valuation`, `periods`, `backtest`, `splits`) hold the
  reference financial logic and the export↔app artifact contract, and are unit-tested.
- The pipeline publishes a small, **schema-asserted** set of Parquet/JSON artifacts to a GitHub
  Release, which every downstream consumer reads.

## Public Streamlit dashboard

- Screener / Company / Backtest pages plus sector surfaces and a valuation-tape hero, fed entirely
  by the published Release artifacts (no Databricks dependency).

## Django web application

- **Read-model architecture** — strict one-directional layering
  (`views → services → repositories → infrastructure → fundamentals_pipeline`); the web app reads
  the published artifacts via **DuckDB** and never queries Databricks at request time. PostgreSQL
  holds only application data.
- **Pages** — company detail (statements + derived metrics), screener (full universe, filters
  narrow it), valuation (football field + margin of safety), and a real landing page + usage guide.
- **Accounts & user data** — custom user model (UUID pk, unique email), session auth
  (login/signup/logout), and named **watchlists**, **favorites**, and browsing **history**.
- **REST API** — versioned DRF endpoints (`/api/v1/companies`, `screener`, `valuation`) over the
  same read model, with pagination, per-IP rate limiting, and an OpenAPI 3 schema + Swagger/Redoc.

## Deployment & operations

- **Production settings** — `DEBUG` off, secrets from the environment, HSTS + secure cookies +
  SSL redirect, WhiteNoise static serving, gunicorn.
- **Fly.io deployment** — multi-stage production image, `release_command` migrations, a `/readyz`
  health gate, and a full runbook (`docs/deploy-fly.md`) including managed-Postgres provisioning.
- **Health & observability** — `/healthz` (liveness) and `/readyz` (readiness: DB + cached
  artifacts) probes; structured JSON logging with a propagated request id; gated Sentry error
  tracking with release tagging; and artifact cache warming with stale-while-revalidate.
- **Security hardening** — API rate limiting, CSRF trusted origins behind the proxy, no
  information disclosure from the readiness probe, and a reviewed auth/injection/IDOR surface.

## Quality, testing & documentation

- **CI gate** — pytest (pipeline + web), mypy + django-stubs, and ruff on every PR; the web suite
  enforces a **coverage floor** (currently ~95%, floor 90%), with integration tests spanning each
  page and the auth/watchlist flows.
- **Documentation** — an architecture contract (`docs/architecture.md`), an **ADR log**
  (`docs/adr/`) recording the significant decisions, and a user-facing help guide.

---

## Data provenance & disclaimer

Figures are derived from **SEC EDGAR filings and market prices** and refresh on the pipeline's run
cadence, so they lag the very latest filing. This is an **informational research tool, not
investment advice**, and may contain errors — verify anything you would act on against primary
sources.

## Known limitations

- **Data freshness** is as-of the last pipeline publish, not real-time.
- **Deployment is prepared, not hosted at tag time** — the Fly.io artifacts and runbook are
  included; provisioning managed Postgres and running `fly deploy` is an operator step.
- **Deferred hardening** (edge rate-limiting for HTML pages, auth brute-force protection, `/admin/`
  exposure) is tracked in #181 and pairs with the production deploy.
- Some **legacy Spanish data labels** (concept/metric hierarchy values) are retained as Delta join
  keys and dashboard display values; new work is English-only.
